Para el modelo agnostico se podria hacer un modelo entidad relaccion normalizado que sea capaz de representar toda la informacion que se 
encuentra en el estandar incial. De esta manera se podria represnetar todas las relacciones de una manera agnostica al lenguaje.

Cerif separa la estructura de los datos y la estructura semantica, lo que permite realziar cambios grandes sin alteral la estrucutra ni realizar migraciones masivas.
El problema es que es muy complejo, se deberia simplificar mucho.


Dado que inicialente se debe crear una manera de representar la informacion agnostica al lenguaje, 
necesitamos algo que nos permita definir toda la informacion antes de pasarlo a Json o python. Existen un gran numero de
maneras de representar ideas agnosticas un lenguaje, sin embargo tras investigar un poco he visto que la mejor opcion, al 
para las fases iniciales, es utilizar Unified Modeling Language (UML) debido a su falilidad de diseño y a que es un lenguaje
que ya he utilizado en otras asignaturas del grado 





PlantUML utiliza texto plano para definir esquemas ulm y devuelve un archivo .puml:
plantuml2pydantic permite pasar plantUML a pydantic
PlantUML-Python-Generator genera clases estandar


Otras herramientas como Enterprise Architect, Visual Paradigm o StarUML exportan en XMI:
PyUML: Una herramienta clásica para mapear archivos XMI a archivos .py
Gaphor: Es una herramienta de modelado UML/SysML escrita en Python que permite, mediante plugins, generar código directamente desde la interfaz.


Despues tenemos:
Visual Paradigm: Cuenta con un motor de ingeniería directa (Forward Engineering) que genera código Python a partir de diagramas de clases de forma nativa.

StarUML: Dispone de una extensión llamada stuml-python que genera esqueletos de clases.
